데스 스트랜딩 2 온 더 비치를 플레이하다 보면 국도 건설이나 모노레일 공사에 자원을 기여해야 할 때가 있다. 금속, 세라믹, 레진 같은 자원을 목표 수량만큼 가져가야 하는데, 문제는 자원을 딱 필요한 만큼 받을 수가 없다는 것이다.

예를 들어 금속은 50, 100, 200, 400 단위로만 수령할 수 있다. 국도 건설에 금속 550이 필요하다면 어떻게 가져가야 할까?

- 400 + 200 = 600 (낭비 50)
- 400 + 100 + 50 = 550 (낭비 0, 하지만 3개)
- 200 × 3 = 600 (낭비 50)

정답은 400 + 100 + 50 = 550으로 낭비가 없는 조합이다. 하지만 낭비가 피할 수 없다면 수령 횟수가 적은 쪽이 낫다. 자원 슬롯을 아낄 수 있으니까.

## 문제 정의

주어진 자원별 수령 단위로, 목표 수량을 맞추거나 초과하는 **최소 낭비 + 최소 단위 수** 조합을 찾아라.

자원별 수령 단위:

| 자원 | 단위 |
|------|------|
| 레진 | 40, 80, 160, 320 |
| 금속 | 50, 100, 200, 400 |
| 세라믹 | 40, 80, 160, 320 |
| 화학물질 | 30, 60, 120, 240 |
| 특수합금 | 60, 120, 240, 480 |

## 그리디의 함정

처음엔 단순 그리디로 짰다. 가장 큰 단위부터 최대한 쓰는 방식. 그런데 문제가 생겼다.

세라믹 390이 필요한 경우:
- 단순 그리디: 320 + 40 + 40 = 400 (3개)
- 최적: 320 + 80 = 400 (2개)

40 두 개 대신 80 하나가 더 낫다. 낭비는 똑같이 10이지만 수령 횟수가 하나 줄어든다.

## 해결 방법

핵심은 **모든 단위가 가장 작은 단위의 배수**라는 관찰이다.

- 레진: 40, 80, 160, 320 → 모두 40의 배수
- 금속: 50, 100, 200, 400 → 모두 50의 배수
- 화학물질: 30, 60, 120, 240 → 모두 30의 배수

그래서 목표값을 최솟값의 배수로 올림한 뒤, 큰 단위부터 그리디로 분해하면 항상 최적해가 나온다.

```python
import math

def solve(need: int, units: list[int]) -> list[int]:
    min_unit = units[0]
    # 목표를 최솟값의 배수로 올림
    target = math.ceil(need / min_unit) * min_unit
    
    # 큰 단위부터 greedy 분해
    result = []
    remaining = target
    for unit in sorted(units, reverse=True):
        while remaining >= unit:
            result.append(unit)
            remaining -= unit
    return result
```

## 만들어보니

Python으로 짜면서 두 가지를 개선했다.

**라벨을 선택적으로** — 처음엔 "국도건설 금속 550" 같은 형식을 강제했는데, 건설 종류 없이 "금속 550"만 입력해도 되도록 바꿨다. 계산 자체에는 건설 종류가 필요 없다.

**여러 자원 동시 계산** — 실제로는 한 번에 여러 자원을 기여한다. "세라믹 390 금속 80" 이렇게 한 번에 입력하면 각각 계산해서 총합을 보여준다.

```
$ python3 calc.py "세라믹 390" "금속 80"

세라믹 390:
  → 320 + 80 = 400 (낭비 10)

금속 80:
  → 100 (낭비 20)

📦 총 수령 목록:
  세라믹 320×1, 80×1
  금속 100×1
```

단순한 계산이지만, 직접 쓰려고 만드니까 더 재밌었다. 게임하면서 손으로 계산하던 걸 스크립트 하나로 해결하는 순간이 역시 제일 뿌듯하다.
