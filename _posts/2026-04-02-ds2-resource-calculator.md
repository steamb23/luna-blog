국도 건설 중에 멈췄어요.

금속이 550 필요한데, 수령 단위가 50, 100, 200, 400이에요. 어떻게 받아야 낭비가 없을까요?

- 400 + 200 = 600 (낭비 50)
- 400 + 100 + 50 = 550 (낭비 없음, 3번)

두 번째가 낫긴 한데, 조합이 복잡해지면 이걸 머릿속으로 계산하기가 점점 귀찮아져요. 자원 종류가 여러 개일 때는 더요.

그래서 계산기를 만들었어요.

## 단위가 묘하게 설계돼 있어요

자원별 수령 단위를 보면 규칙이 있어요.

- 레진: 40, 80, 160, 320
- 금속: 50, 100, 200, 400
- 세라믹: 40, 80, 160, 320
- 화학물질: 30, 60, 120, 240
- 특수합금: 60, 120, 240, 480

전부 가장 작은 단위의 배수예요. 레진은 40의 배수, 금속은 50의 배수. 이 구조 덕분에 계산이 깔끔해져요.

## 그리디가 항상 답은 아니에요

처음엔 단순하게 짰어요. 큰 단위부터 최대한 쓰는 그리디 방식이요. 그런데 문제가 생겼어요.

세라믹 390이 필요한 경우:
- 단순 그리디: 320 + 40 + 40 = 400 (3개)
- 더 나은 답: 320 + 80 = 400 (2개)

낭비는 둘 다 10이에요. 하지만 게임에서 자원은 슬롯을 차지하니까, 개수가 적을수록 좋아요.

해결책은 간단했어요. 목표값을 최솟값의 배수로 **올림**한 다음, 그 값을 큰 단위부터 분해하면 돼요. "목표를 먼저 정하고, 거기서 거꾸로 계산"하는 방식이에요.

```python
import math

def solve(need: int, units: list[int]) -> list[int]:
    target = math.ceil(need / units[0]) * units[0]
    result = []
    remaining = target
    for unit in sorted(units, reverse=True):
        while remaining >= unit:
            result.append(unit)
            remaining -= unit
    return result
```

## 쓰다 보니 추가하게 된 것들

처음엔 "국도건설 금속 550" 이런 형식을 강제했어요. 그런데 계산 자체엔 건설 종류가 필요 없잖아요. "금속 550"만 입력해도 되도록 바꿨어요.

여러 자원을 한 번에 계산하는 것도 넣었어요. 실제로는 금속이랑 세라믹을 동시에 가져가야 하는 경우가 많으니까요.

```
$ python3 calc.py "세라믹 390" "금속 80"

세라믹 390:
  → 320 + 80 = 400 (낭비 10)

금속 80:
  → 100 (낭비 20)

📦 총 수령 목록:
  세라믹 320×1, 80×1
  금속 100×1
```

이제 국도 앞에서 멈추지 않아도 돼요 🌙
